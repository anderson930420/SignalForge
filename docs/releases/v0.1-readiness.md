## Artifact Contracts

### signal.csv
- **Status:** PASS
- **Schema:** `datetime, available_at, symbol, signal_name, signal_value, signal_binary, source`
- **Deterministic:** Rows sorted by `datetime, symbol, signal_name`
- **Rules:** `available_at <= datetime`, `signal_binary in {0, 1}`, no duplicates
- **Evidence:**
  - `src/signalforge/schemas.py:SIGNAL_COLUMNS`
  - `tests/test_signal_artifact_contract.py`
  - `tests/test_e2e_generate_artifacts.py`

### signal_contract.yaml
- **Status:** PASS
- **Schema:** Nested canonical format with `factor.name`, `factor.version`, `factor.parameters`, `decision_rule.signal_binary`, `timing.available_at_rule`, `output.file`, `output.schema_version`, `output.columns`
- **Deterministic:** No `exported_at` timestamps; parameters sorted alphabetically
- **Evidence:**
  - `src/signalforge/export.py:build_signal_contract()`
  - `tests/test_export.py:TestDeterministicExport`
  - `tests/test_e2e_generate_artifacts.py`

### data_quality_report.json
- **Status:** PASS
- **Schema:** Source-OHLCV focused with `missing_values` per column, `point_in_time_correctness_claimed: false`
- **Deterministic:** No volatile timestamps; warnings sorted by code
- **Evidence:**

## Known Limitations

1. **Single factor:** Only Moskowitz time-series momentum is implemented. Future factors require additional implementation.
2. **Single symbol per run:** The CLI generates signals for one symbol at a time.
3. **CSV only:** No database or streaming data source in MVP.
4. **Daily frequency only:** Intraday signals not supported in MVP.
5. **No parameter validation:** Factor parameters are not validated against expected ranges in CLI.
6. **No custom factor registration via CLI:** Factors are hard-coded in registry.

---

## Suggested Next Milestones

1. **Additional factors:** Add more factor implementations (e.g., value factor, quality factor)

3. **Data source abstraction:** Support database or API-based data loading

5. **Configuration file schema:** Formal YAML schema validation
